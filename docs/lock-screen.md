# 锁屏模块设计

## 适用范围
- 仅支持 Obsidian 桌面端（`isDesktopOnly: true`）。
- 为工作区提供访问授权码保护。
- 在操作系统锁屏时自动锁定。
- 支持可选的启动即锁定策略。

## 设计目标
- 在系统锁屏后，确保笔记内容不可见。
- 在输入正确授权码前，禁止查看与编辑。
- 避免以明文形式存储授权码。

## 非目标
- 不支持移动端。
- 不包含云同步或账号体系。
- 不包含生物识别解锁。

## 核心模块
- `src/main.ts`
  - 负责插件生命周期管理。
  - 加载与保存插件设置。
  - 注册事件监听与命令。
- `src/settings.ts`
  - 提供设置页交互。
  - 支持授权码设置、更新与清除。
- `src/security.ts`
  - 提供授权码哈希与校验能力。
  - 采用 PBKDF2 + SHA-256 + 随机盐。
- `src/lockManager.ts`
  - 负责锁定状态机。
  - 处理锁定触发与解锁流程。
  - 锁定时拦截全局快捷键绕过。
- `src/overlay.ts`
  - 渲染全屏不透明遮罩。
  - 处理解锁输入与错误提示。

## 设置模型
- `enabled`：是否启用授权码保护。
- `lockOnSystemLock`：系统锁屏时是否自动锁定（默认 true）。
- `lockOnStartup`：启动时是否立即锁定（默认 false）。
- `passcodeSalt`：base64 编码的随机盐。
- `passcodeHash`：base64 编码的哈希值。
- `hashIterations`：PBKDF2 迭代次数（默认 210000）。

## 锁定与解锁流程
1. 插件启动并加载设置，注册事件监听。
2. 操作系统触发锁屏事件。
3. `LockManager` 判断功能启用且已配置授权码。
4. 渲染全屏不透明遮罩并进入锁定态。
5. 用户输入授权码并提交。
6. 基于盐和迭代参数重新计算哈希并做常量时间比较。
7. 校验成功则移除遮罩，恢复正常访问。
8. 校验失败则保持锁定并提示错误。

## 多窗口同步策略
- 使用 `BroadcastChannel` 在多个窗口间广播锁定/解锁事件。
- 使用 `localStorage` 记录全局锁定状态，保证新开窗口可立即继承锁定态。
- 增加基于插件数据文件的轮询兜底同步（`syncStateVersion` + `syncLocked`）。
- 任一窗口锁定后，其他窗口同步进入锁定。
- 任一窗口解锁后，其他窗口同步解锁。
- 当 `BroadcastChannel` 或 `storage` 事件不可用时，仍可通过数据文件轮询完成同步。

## 第三方窗口豁免策略
- 支持对特定窗口设置“锁屏豁免”，用于快捷采集等场景。
- 豁免窗口在全局锁定时保持可输入、可保存，不弹出授权码页面。
- 豁免窗口不会改变主窗口/副窗口的锁定状态。
- 支持以下任一标记方式：
  - `window.__LOCK_SCREEN_EXEMPT__ = true`
  - `document.documentElement.setAttribute("data-lockscreen-exempt", "true")`
  - `document.body.setAttribute("data-lockscreen-exempt", "true")`
- 若取消豁免（改为 `false` 或移除属性）且当前处于全局锁定态，该窗口会自动回到锁定。

## 窗口控制策略
- 锁屏遮罩会避开顶部窗口控制区域，不覆盖最小化/还原/关闭按钮。
- 仍保持编辑区内容完全遮挡，不透出笔记正文。
- 支持点击最小化按钮时隐藏到托盘；关闭按钮保持正常退出行为。

## 安全策略
- 不持久化明文授权码。
- 使用随机盐与高迭代 PBKDF2 增强抗暴力破解能力。
- 使用常量时间比较降低时序侧信道风险。
- 锁定态使用完全不透明遮罩，避免底层内容泄露。
- 锁定态下拦截全局按键，降低快捷键绕过风险。

## 交互策略
- 锁屏面板居中展示，信息简洁。
- 输入框支持回车提交。
- 设置页和命令面板均可触发“立即锁定”。
- 授权码最小长度为 4。
- 锁屏界面配色与 Obsidian 当前浅色/深色主题变量保持同步。

## 当前限制
- 部分运行环境可能无法访问 Electron 锁屏事件通道。
- 尚未实现失败次数限制与冷却策略。

## 后续规划
- 增加连续失败后的冷却机制。
- 增加可配置的自动锁定延迟。
- 评估并接入 Electron 层系统锁屏事件。

## 最近更新

- 新增强制焦点守卫，锁定态持续确保输入焦点可用。
- 调整全局键盘拦截策略：仅阻断遮罩外输入与高风险组合键，保留授权码输入流畅性。
- 自动锁定触发策略改为仅系统锁屏事件，不再因窗口切换触发。
- 新增跨窗口锁定状态同步：一处解锁，全部窗口同步解锁。
- 修复部分环境下仅主窗口锁定的问题，新增磁盘轮询同步兜底机制。
- 锁屏遮罩调整为不覆盖顶部窗口控制区域，支持最小化/还原/关闭。
- 修复部分窗口已锁定但遮罩不可见的问题，为锁屏界面补充关键内联样式兜底。
- 修复“副窗口未显示授权码输入页”的问题：改为按窗口实例创建独立锁屏管理器与遮罩。
- 新增工作区窗口跟踪（`window-open` / `window-close`），确保新开窗口立即纳入锁屏控制。
- 修复副窗口授权码无法输入的问题：改进跨窗口按键命中判断并补充字符回填逻辑。
- 进一步修复副窗口输入焦点异常：当按键来源不在遮罩内时，仍可将字符/退格回填到授权码输入框。
- 修复同进程多窗口解锁不同步的问题：任一窗口解锁后会立即同步到其余窗口。
- 新增第三方窗口豁免能力：可指定快捷采集窗口不锁屏且不影响其他窗口锁屏状态。
- 集成快捷记录模块，弹窗默认自动设置锁屏豁免并支持 `Ctrl+S` 保存后自动关闭。
- 修复锁屏回归问题：手动锁定与系统锁屏触发后会正确写入全局锁定状态，不再被策略巡检误解锁。
- 修复速记弹窗输入串扰问题：锁屏焦点守卫仅在窗口自身激活时生效，避免后台主窗口抢焦点。
- 调整快捷记录唤起行为：打开或激活速记窗口时不再强制唤起主窗口。
- 重构授权码页面视觉风格并接入主题变量，浅色/深色模式自动同步。
- 修复速记窗口聚焦体验：打开后自动聚焦编辑输入区域，可直接键入内容。
- 修复锁定主窗口被速记弹窗激活后自动解锁的问题：窗口聚焦时按静默策略重算锁定状态。
- 调整速记窗口策略：不再主动隐藏主窗口，避免前台主窗口消失。
- 新增插件内置 Electron 全局热键注册能力，用于绕开 `Global Hotkeys` 带来的主窗口抢焦点问题。
- 优化速记窗口关闭收尾：抑制已知 `ResizeObserver` 控制台告警，减少误报。
